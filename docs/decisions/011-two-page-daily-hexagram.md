# 011 今日卦象采用两页式连续阅读

- 状态：已确认，本地验证通过，待发布确认
- 日期：2026-07-24
- 关联 Issue：#23

## 决定

“今日卦象”只使用两个页面状态：

```text
首页今日一卦入口
    ↓ 一次点击
今日卦象完整结果页
```

结果页内部以纵向分区连续展示日期与主题、卦象结构、经典依据、现代观察提示、理性边界和知识页入口。不得把这些内容拆成步骤页，不使用“下一步”，也不要求人物档案或填写表单。

## 每日卦象口径

- 首页卡片与结果页共用一个纯函数，按访客本地公历日期稳定选择 1–64 卦。
- 同一自然日必须得到同一卦；跨过访客本地午夜后自动更新。
- 2026-07-24 保持现有视觉基线，映射为第 62 卦“雷山小过”。
- 这一日期轮换是产品的文化阅读机制，不冒充蓍草、铜钱、梅花易数或其他传统起卦法；页面必须公开说明。

## 内容边界

- 卦辞与《象》曰读取项目已核对的《周易》数据，标注为经典原文。
- 今日提示单独标注为现代辅助解释，使用观察问题与可复盘的小行动，不输出确定性预测。
- 不替代医疗、法律、投资或其他现实决策。
- 结果页只提供一个进一步入口：进入对应的六十四卦知识页。

## 视觉与组件

- 复用现有昼夜主题、二级页页眉、宋楷标题、米纸或深墨静态背景和古金交互。
- 今日结果页不运行 LightRays 或 LiquidEther；WebGL 仍只属于首页和六十四卦知识页的可见 Banner。
- 二级页页眉与主题切换抽成共享组件，避免知识页和今日页各维护一套逻辑。
- 移动端使用自然纵向滚动，支持最小 320px 且不产生横向溢出。
