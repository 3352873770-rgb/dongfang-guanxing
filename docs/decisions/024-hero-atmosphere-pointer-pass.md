# 024：首页 Hero 背景指针入场

- 状态：已采纳
- 日期：2026-07-24

## 背景

首页昼间 LightRays 与夜间 LiquidEther 都能响应桌面鼠标。用户希望首次进入时主动展示一次这种响应，但不希望按钮本身移动，也不希望把演示误做成按钮描边扫光。

## 决定

桌面端首次进入首页后，以“开始问卦”按钮中心的 Y 坐标作为水平轨迹，在 Banner 内派发一次 900ms 的虚拟鼠标左、右、左往返。事件继续交给当前已挂载的 LightRays 或 LiquidEther 处理，因此昼夜共用一套触发逻辑，不新增第三套背景动画。

轨迹只执行一次，不显示虚拟光标，不改变 CTA 的位置或 BorderGlow。移动端、粗指针设备和 `prefers-reduced-motion` 环境直接跳过。

## 原因

复用真实指针响应能够准确展示现有背景特效，并保持昼夜组件、性能边界和交互语义不变。以 CTA 高度定位轨迹可以把视觉能量集中在首屏主操作附近，同时避免把动画误解为按钮位移。

## 回滚

移除首页 Hero 中派发合成 `mousemove` 的一次性 Effect 即可；LightRays、LiquidEther、CTA BorderGlow 与其他交互无需修改。
