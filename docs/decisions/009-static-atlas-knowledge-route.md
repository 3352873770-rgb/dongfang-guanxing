# 009：首页静态图谱与六十四卦知识路由

- 状态：已采纳
- 日期：2026-07-24

## 背景

首页六十四卦图谱原本使用水平轨道、箭头和分页点。移动端虽然能看到部分卦象，但横向浏览会隐藏内容边界，也无法承载完整的六十四卦科普。

## 决定

首页 `#atlas` 改为响应式静态网格，移除水平滚动、轮播箭头和分页点。每个卦象入口都通过渐进增强获得鼠标、键盘和触摸交互，并进入 `#/hexagrams/:number` 知识页。

知识页采用 Hash 路由以兼容 GitHub Pages，页面包含阅读方法、当前选中卦象和可搜索的六十四卦总览。二级页继续使用昼夜双主题：夜间为深墨与低强度 LiquidEther，日间为暖米纸与低强度 LightRays。

## 影响

- 首页图谱信息边界更明确，移动端不再横向滑动。
- 六十四卦知识按需加载，不增加首页首屏主包内容。
- 科普内容强调阅读方法和文化知识，不把卦象包装成现实决策的固定答案。
- 本任务默认停在本地验证与草稿 PR，等待用户单独确认公网发布。
