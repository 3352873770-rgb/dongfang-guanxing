# 007 开始观星问卦引导流程

- 状态：已完成本地实现与浏览器验证
- 日期：2026-07-24
- 关联 Issue：#17

## 决定

“开始观星问卦”使用单页完整输入表单，不在步骤之间跳回首页：

```text
选择所问方向、写下具体问题、选择或新建人物档案
    ↓
保存档案并起卦
    ↓
展示卦象结果
```

首页 Hero 的“开始观星问卦”进入完整表单。“长期问卦”中的感情发展、事业发展、学业考试和财富运势按钮仅预选对应方向，仍展示完整表单。返回修改信息时保留已选择的方向、问题和档案信息。

表单页与结果页的页眉统一使用“·观星问卦 ·”，不显示步骤或阶段提示。

## UX 设计理由

本流程遵循“用信息连续替代步骤堆叠”的设计理念。用户的真实目标是完成一次问卦，而不是依次完成多个页面；因此，所问方向、具体问题和人物档案应作为一次连续任务呈现。

单页表单让用户在提交前看见完整流程并自由检查信息，减少“下一步”带来的额外点击和上下文切换。进入结果页是唯一必要的页面转换，因为此时任务状态已经从“准备问卦”变为“阅读卦象”。

简化流程不等于减少必要信息。分类、问题和档案仍然完整保留，只是通过章节、留白和分隔线组织在同一页面。返回修改时保留原有内容，避免用户重复填写。

## 首期分类

- 感情发展
- 事业发展
- 学业考试
- 财富运势
- 其他所问

问题输入必须围绕一件具体事情。界面提示用户补充处境和希望看清的变化，不将问卦包装为医疗、法律、投资或现实决策的替代。

## 起卦与结果

用户完成分类、问题和人物档案后，通过唯一主操作“保存档案并起卦”直接生成卦象结果。系统不再增加起卦方式选择页，也不在结果前插入额外确认步骤。
