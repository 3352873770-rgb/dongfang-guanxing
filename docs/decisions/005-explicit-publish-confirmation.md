# 005：公网发布必须逐任务确认

- 状态：Accepted
- 日期：2026-07-22

## 背景

项目会持续进行多轮设计、开发和优化。如果每次任务完成后都自动合并并更新 GitHub Pages，用户将失去对公网版本发布时间和变更组合的控制。

## 决策

每项任务默认只完成本地验证、提交分支并创建草稿 PR。只有用户针对当前任务明确确认“发布”后，才允许将 PR 转为可合并状态、合并 `main` 并触发 GitHub Pages。

发布确认只对当前任务有效，不延续到后续任务。版本标签和 GitHub Release 也需要当次明确授权。当前 PR #16 已获得用户授权，作为本规则生效前需要完成的最后一次发布；该次发布完成后，所有新任务执行本门禁。

## 影响

- 用户可以决定每次公网更新的时机。
- 已完成但未获发布确认的任务会保留在分支和草稿 PR 中，不影响线上版本。
- 任务交付时需要明确报告本地验证与 PR 状态，并等待用户确认，而不是自动上线。
- 回滚方式不变，仍通过 `git revert` 和独立 PR 保持 `main` 历史可追溯。
