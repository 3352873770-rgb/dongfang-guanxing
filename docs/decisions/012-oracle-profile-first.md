# 012 灵签信五工具统一以人物档案开场

- 状态：已确认，本地验证通过，待发布确认
- 日期：2026-07-24
- 关联 Issue：#26

## 决定

“云签解惑”“事业灵签”“流年运势”“时辰运势”“AI 解读报告”五个二级输入页统一采用以下顺序：

```text
工具标题与用途说明
    ↓
人物档案共享组件
    ↓
工具专属字段
    ↓
唯一主操作
    ↓
直接结果
```

人物档案组件必须处于第一个表单位置，包含已有档案选择、新建档案、可编辑出生信息、广州 11 区与经度调整。五个工具复用同一组件和本地存储，不复制字段逻辑。

## 数据与算法边界

- 所有工具生成结果前保存并关联当前档案，便于区分人物和归档记录。
- 流年运势可按其既有定义使用人物背景。
- 云签解惑、事业灵签、时辰运势与 AI 解读报告只保存档案归属，不把出生日期偷偷加入原有起卦或报告算法。
- 结果仍需明确传统口径、现代解释与理性边界，不输出确定性预测。

## UX 边界

- 保持“单页填写 → 直接结果”，不增加独立档案页或步骤条。
- 档案后直接进入当前工具字段，不返回首页，不丢失入口上下文。
- 页面继续只有一个主操作；档案保存随生成结果自动完成。
