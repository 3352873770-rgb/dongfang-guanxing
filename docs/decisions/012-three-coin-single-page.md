# 012 三枚铜钱采用单页完整任务流

日期：2026-07-24  
状态：已确认设计方向，正在本地验证，待发布确认

## 决策

“三枚铜钱”作为推荐小工具的首个完整二级页，采用一个连续页面完成“01 定问 → 02 六掷 → 03 见卦”。

- 事项分类、具体问题、六次三钱记录和结果都保留在同一页面。
- 六次记录自初爻至上爻；每次三枚铜钱按“正面记 3、反面记 2”生成 6、7、8、9 四种爻值。
- 六次记录未完成时不能生成结果；完成后结果直接在当前页下方展开。
- 结果区分经典依据、现代行动建议、风险提醒和理性边界，不输出确定性预测。
- 页面复用现有二级页页眉、昼夜主题和排版语言，不新增大面积插画。

## 原因

该方案延续项目“用信息连续替代步骤堆叠”的 UX 理念，减少页面跳转与上下文丢失，同时让用户能够在结果出现后继续对照自己的问题和完整起卦记录。

## 后果

- 其他推荐工具应优先复用同一单页骨架，只替换必要输入、玩法组件和结果结构。
- 页面可能较长，但移动端允许自然纵向滚动；不能通过增加步骤页来掩盖内容长度。
- 本功能默认停在本地验证与草稿 PR，只有用户明确确认本次发布后才更新 GitHub Pages。
