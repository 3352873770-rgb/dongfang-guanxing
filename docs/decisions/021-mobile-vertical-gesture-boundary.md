# 021 移动端纵向手势边界

- 状态：已采纳
- 日期：2026-07-24

## 决定

在既有根节点横向裁切之外，进一步建立内容级手势边界：

- `#ask` 与 `#tools` 仅接受纵向页面手势；
- 720px 以下将问卦和工具按钮的 SpecularButton 横向特效缓冲及 WebGL 画布显示宽度收进各自按钮宽度，覆盖渲染器运行时写入的内联尺寸；
- 人格偏好、六十四卦、今日卦象与三枚铜钱四个二级页统一使用 `overscroll-behavior-x: none` 与 `touch-action: pan-y`；
- 顶部导航仍保留自身的局部横向滚动能力。

## 原因

根页面的 `scrollWidth` 即使等于视口宽度，按钮内部超出边界的 WebGL 描边画布仍可能让部分移动浏览器产生横向回弹或拖拽感。将特效缓冲限制在单个按钮并声明纵向手势，可以保留逐按钮响应，同时消除页面级横向反馈。

## 回滚

若后续交互需要横向手势，应只在明确的局部控件重新开放，不应移除整个页面的纵向手势边界。回滚时需重新验证 320px、351px、390px 与 476px 视口下的 `scrollWidth`、`scrollX` 和逐按钮特效。
