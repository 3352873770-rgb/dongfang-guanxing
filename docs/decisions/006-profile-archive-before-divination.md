# 006 问卦前人物档案

- 状态：已完成本地实现与视觉校验
- 日期：2026-07-24

## 决定

用户进入正式起卦前，必须先选择已有档案或新建人物档案。档案选择不再使用单独页面，而是整合到出生信息表单顶部；档案保存后可重复使用，并可通过独立档案入口查看、编辑和删除。

## 功能入口逻辑

所有依赖人物出生信息的功能共用同一道档案门禁：

```text
用户点击功能
    ↓
系统检查本地档案
    ├─ 没有档案
    │    ↓
    │  打开档案信息表单
    │    ↓
    │  下拉菜单显示新建档案，表单为空
    │    ↓
    │  填写并保存，进入原功能
    │
    └─ 已有档案
         ↓
       打开同一档案信息表单
         ↓
       顶部下拉默认选中最近档案
         ↓
       自动回填下方信息
         ↓
       用户确认或修改，进入原功能
```

系统在档案建立或选择期间必须保留用户最初点击的功能、问题分类和其他已选上下文。完成档案步骤后直接继续原功能，不能返回首页让用户重新寻找入口。

“选择档案”位于“基本信息”上方。即使当前只有一个档案，也在下拉菜单中显示该档案供用户确认，并提供“新建档案”入口。用户切换档案后，姓名、性别、历法、出生日期、出生时间、时辰、地区和经度必须整体回填，且允许继续编辑。档案保存失败、表单未完成或用户主动取消时，不得进入后续起卦流程。

首期档案字段包括：

- 姓名
- 性别
- 出生日期
- 历法切换：公历 / 农历
- 出生时间与自动匹配的十二时辰
- 省 / 市 / 区三级所在地
- 根据地区自动匹配的经度
- 手动修改经度入口

公历与农历是同一个出生日期的两种输入口径，不保存为两个互相独立的日期。地点数据后续按全国行政区划扩展，首期以广州为例。

## 广州首期区级数据

广州当前包含越秀区、海珠区、荔湾区、天河区、白云区、黄埔区、花都区、番禺区、南沙区、从化区、增城区等 11 个区。区级范围以[广州市人民政府行政区域资料](https://www.gz.gov.cn/zlgz/gzgk/xzqy/)为依据。

地区自动经度只作为输入辅助，界面必须说明“自动匹配，可修改”，不能把区级中心点等同于用户精确出生地点。
