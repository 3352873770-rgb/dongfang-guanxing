# Design System

## 品牌原则

- 气质：东方、克制、安静、可信。
- 夜间：深墨背景、古金文字、LiquidEther 流体氛围。
- 日间：暖米纸背景、茶棕文字、日照旧金、LightRays 长体积光。
- 两种模式共享相同结构、文案层级和交互逻辑。

## UX 设计理念：用信息连续替代步骤堆叠

东方观星以“简化用户完成目标的路径”为核心 UX 原则。简化不是删去必要信息，而是重新组织信息，减少没有业务必要的点击、页面切换和上下文中断。

- 可以在同一任务界面连续完成的内容，优先使用清晰分区的单页流程，不把每一个填写动作拆成独立步骤。
- 观星问卦的主流程固定为“观星问卦完整表单 → 起卦结果”。所问方向、具体问题、人物档案和出生信息在同一页面完成，用户只在真正生成结果时进入下一页面。
- 长页面滚动优先于反复点击“下一步”。使用章节标题、留白和分隔线建立阅读节奏，让用户随时看见流程全貌。
- 每个主任务只保留一个清晰的主要操作。观星问卦表单以“保存档案并起卦”为唯一主操作，不在中途插入竞争性的继续按钮。
- 入口可以预选内容，但不能跳过或隐藏必要信息。“长期问卦”只预选所问方向，完整表单仍然可见、可检查、可修改。
- 同类任务共享页面骨架和组件，不通过复制页面制造差异。“灵签信”五个二级工具复用长期问卦的页眉、宣纸容器、章节节奏、输入控件和唯一主操作；差异集中在必要字段与结果信息层级。
- 人物档案作为共享组件维护。“灵签信”五个二级工具都把档案选择、新建与编辑放在工具专属字段之前，生成结果时自动保存并关联当前档案。
- 档案在非出生类工具中只承担人物识别与记录归属，不得暗示出生日期参与了云签、事业灵签、时辰运势或 AI 报告的原有计算口径。
- 返回修改时必须保留已经选择和填写的内容，避免重复输入；错误提示应帮助用户修正当前信息，而不是把用户送回流程起点。
- 桌面端和移动端共享同一信息架构。移动端允许自然纵向滚动，但不得通过增加页面层级来掩盖内容长度。
- 只有在权限确认、高风险操作、复杂条件分支或单页认知负担明显过高时，才增加独立步骤，并在设计文档中说明必要性。

这套理念的衡量标准是：更少的无效点击、更少的页面切换、更少的重复输入，以及更完整的任务上下文。

## 字体

- 展示标题：`Kaiti SC`、`STKaiti`、`KaiTi`、`Songti SC`。
- 正文与界面：`Noto Sans SC`、`PingFang SC`、系统无衬线字体。
- 标题避免拥挤，正文保持充足行高与移动端可读性。

## 色彩语义

- 夜间页面：深墨青 `#071416` 附近。
- 夜间强调：古金 `#c29b61`、亮金 `#e0bb7d`。
- 日间页面：暖米纸 `#f4efe6` 附近。
- 日间强调：茶棕 `#805627`、深旧金 `#674116`。
- 高光应服务层级，不使用纯亮黄或大面积发光。

## 布局

- Hero 始终保持居中编辑式构图。
- 一级内容使用稳定纵向节奏和清晰分区。
- 移动端以 390×844 为主要验证视口，同时支持最小 320px。

## 动效

- 经典书名保持错峰横向或纵向漂移，周期约 9.4–12.8 秒。
- `#ask` 与 `#tools` 必须逐按钮响应，禁止整区共享高光。
- 按钮可使用描边流光、图标轻抬、文字提亮和箭头位移。
- 日间动效使用低强度茶棕和旧金；夜间使用古金。
- reduced-motion 下关闭持续动画并保留静态可读状态。
- LightRays 与 LiquidEther 只允许在首页和六十四卦知识页的 Banner 容器内渲染；Banner 离开视口或页面进入后台后卸载 WebGL，下方内容只使用静态主题底色。

## 今日卦象按钮

- `#daily` 卡片的主操作统一显示“今日卦象”，使用旧金到暖金的圆角矩形填充，不使用弱描边式次级按钮。
- 移动端触摸高度不低于 44px；通过金色填充、深墨文字、短箭头以及 hover、focus-visible、active 状态明确表达可点击性。
- 按钮视觉增强仅作用于本 CTA，不扩散为卡片或整个区块的共享高光。

## 今日卦象结果页

- 路径固定为“首页今日一卦 → 今日卦象完整结果页”，不增加步骤页、“下一步”、人物档案或表单门槛。
- 首页与结果页共用按访客本地公历日期轮换的卦象数据；同一自然日的日期、卦名、主题和卦象必须一致，并在本地午夜后自动更新。
- 结果页使用连续纵向阅读：今日主题、卦象结构、经典依据、现代观察、理性边界和对应六十四卦知识入口。
- 经典原文与现代辅助解释必须明确分区；页面公开说明日期轮换不是传统起卦方式，现代解释不输出确定性预测。
- 复用昼夜主题与二级页共享页眉，但结果页不运行 LightRays 或 LiquidEther；移动端自然滚动且最小 320px 不横向溢出。

## 六十四卦图谱

- 首页 `#atlas` 使用静态响应式网格，不使用水平滚动、轮播箭头或分页圆点。
- 首页卦象元素必须明确可点击，并支持鼠标、键盘和触摸；点击后进入 `#/hexagrams/:number` 知识页并聚焦对应卦象。
- 知识页采用“阅读方法 → 当前卦象 → 六十四卦总览”的连续信息结构，以纵向阅读代替多层跳转。
- 夜间 Banner 使用深墨、古金与低强度 LiquidEther；日间 Banner 使用暖米纸、茶棕、日照旧金与低强度 LightRays。两种模式保持相同结构、内容密度和交互，阅读区与图谱区不持续运行 WebGL。
- 六十四卦网格桌面端最多四列，移动端两列，最小 320px 可降为单列；任何视口不得产生横向溢出。
- 内容定位为《周易》文化知识与阅读线索，不以卦象替代现实判断，也不输出绝对化预测。

## 可访问性

- 所有按钮提供可见 `focus-visible`。
- 文字和背景保持足够对比。
- 触摸目标不依赖 hover 才能理解。
- 状态变化使用可理解的标签，而非只依赖颜色。
