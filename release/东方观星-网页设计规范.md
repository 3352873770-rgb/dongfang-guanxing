# 东方观星｜网页设计规范

版本：V1.0

适用范围：一级首页、移动端自适应、Figma 交付、前端开发与离线分享版

基准实现：React + Vite 当前生产页面

## 1. 设计定位

东方观星是一套以《易经》“观象、明理、趋时、知变”为核心的东方决策体验。视觉不制造恐惧，不使用廉价玄学符号堆叠，而是通过深青夜色、旧金、宣纸、山水和留白建立“可信、克制、可沉浸”的心理感受。

设计关键词：东方、观象、夜色、纸感、秩序、留白、理性自省。

用户情绪路径：好奇进入 → 选择问题 → 获得仪式感 → 理解变化 → 形成行动建议 → 留存记录。

## 2. 页面信息架构

1. 顶部导航：品牌、首页、问卦、卜卦、记录、我的、AI 解读报告。
2. 首屏观象：循环视频、主标题、问题主题、主要行动按钮、使用指南。
3. 长期问卦与灵签信：高频用户问题入口。
4. 今日一卦：每日内容与卦象解释入口。
5. 人格图谱：自我认知玩法和测试入口。
6. 解读价值：变、理、行、果四层解释结构。
7. 六十四卦图谱：横向浏览与选择。
8. 推荐小工具：八种易学工具入口。
9. 问卦记录：记录、追踪、前后对比。
10. 理性提示：不替代现实决策，不鼓励迷信。

## 3. 画布、栅格与间距

### 3.1 Figma 画布

| 场景 | Frame 宽度 | 内容容器 | 页面边距 |
| --- | ---: | ---: | ---: |
| Desktop | 1440 px | 最大 1180 px | 最小 64 px |
| Tablet | 900 px | 自适应 | 32 px |
| Mobile | 390 px | 362 px | 14 px |
| 最小支持宽度 | 320 px | 292 px | 14 px |

桌面端采用 12 栏思路组织内容；双栏模块比例按内容调整，不强制平均。移动端统一改为单列，工具卡保持双列。

### 3.2 间距体系

基础单位为 4 px，优先使用：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64。

- 一级区块垂直间距：桌面 30–52 px，移动 24–34 px。
- 主内容容器：最大宽度 1180 px。
- 常规卡片内边距：桌面 22–64 px，移动 16–20 px。
- 卡片圆角：14 px；工具卡 12 px；日卦卡 10 px；主要按钮 8–14 px。
- 描边：1 px，使用低透明度旧金色，避免高亮金边过多。

## 4. 色彩规范

| Token | 色值 | 用途 |
| --- | --- | --- |
| Page / 夜青 | `#071416` | 页面主背景 |
| Surface | `rgba(14,27,28,.72)` | 常规深色卡片 |
| Surface Strong | `rgba(16,29,29,.90)` | 强调浮层与导航 |
| Gold | `#C29B61` | 标准旧金 |
| Gold Bright | `#E0BB7D` | 标题、选中、主图标 |
| Gold Muted | `#9C7D54` | 次级图标与弱信息 |
| Ivory | `#E6CCAA` | 主标题和高层级正文 |
| Paper | `#DDC5A0` | 宣纸按钮与浅色卡片 |
| Ink | `#202828` | 浅色卡片上的文字 |
| Muted | `#A58F70` | 说明文字 |
| Line | `rgba(190,151,94,.32)` | 标准描边 |
| Line Soft | `rgba(190,151,94,.17)` | 分割线、弱描边 |

使用规则：单个视区内高亮金色面积不超过约 15%；正文优先使用象牙白或灰金，不用纯白；深色卡片之间依靠明度和边框区分，不使用强烈投影。

## 5. 字体规范

字体栈：

- 标题与东方叙事：`Noto Serif SC` / `Songti SC` / `serif`。
- 功能文字与辅助信息：`Noto Sans SC` / `PingFang SC` / `Microsoft YaHei` / `sans-serif`。

| 层级 | Desktop | Mobile | 字重 | 行高 / 字距 |
| --- | ---: | ---: | ---: | --- |
| Hero H1 | 36–60 px | 35–43 px | 500 | 1.18 / 0.09–0.15em |
| 大模块标题 | 34–58 px | 38 px | 500 | 1.15 / 0.14em |
| 标准模块标题 | 20–29 px | 20 px | 500 | 1.4 / 0.10–0.16em |
| 卡片标题 | 13–20 px | 14–15 px | 500 | 1.5 / 0.07em |
| 正文 | 12–16 px | 10–12 px | 400 | 1.65–1.9 / 0.04–0.08em |
| 辅助文字 | 9–13 px | 8–10 px | 400 | 1.5–1.65 / 0.03em |

中文标题允许适度字距营造仪式感；较长正文不得使用超过 0.08em 的字距。按钮标题保持一行，说明文字保持一行并在必要时省略。

## 6. 核心组件规范

### 6.1 顶部导航

- 桌面高度 62–88 px；品牌位于左侧，主导航绝对居中，报告入口位于右侧。
- 当前项文字使用 Gold Bright，并显示 24 px × 1 px 下划线。
- 760 px 及以下切换为右侧菜单按钮；打开后使用全屏深青抽屉。
- 移动菜单单项最小高度 57 px，关闭按钮点击区 42 × 42 px。

### 6.2 观象视频圆盘

- Desktop 直径 320–604 px；Mobile 宽度 `min(83vw, 340px)`。
- 三层旧金描边，外层 1 px，保持克制。
- 视频填满正圆，`object-fit: cover`，定位 50% / 48%，视觉缩放 1.13。
- 叠加 `sepia(.21) saturate(.72) contrast(.92) brightness(1.02)`。
- 播放按钮位于底部 8% 中央，尺寸 27–38 px。

### 6.3 主题胶囊

- 高度 27–36 px，移动端 29 px。
- 描边 1 px，圆角 999 px；相邻按钮允许 -1 px 合边。
- Default：灰金文字；Hover / Selected：亮金文字、亮金边、12% 金色底。
- 选中状态必须同时提供 `aria-pressed=true`，不能只依靠颜色。

### 6.4 主行动按钮

- Desktop：最大 660 px，宽度约 46vw，高度 48–70 px。
- Mobile：宽 208–248 px，约 60vw，高 42 px。
- 宣纸底色叠加山水纹理；文字深墨色。
- Hover：整体上移 4 px并轻微提亮；Active：上移 1 px并缩放至 0.985。
- Loading：按钮禁用，页面出现问卦路径生成浮层。

### 6.5 卡片

- 深色卡片背景透明度控制在 66%–76%。
- 常规边框为 Line / Line Soft；Hover 不改变布局尺寸。
- 工具卡桌面四列，900 px 以下两列；移动端单卡最小高度 136 px。
- 图标优先使用 Phosphor 线性图标，`weight=thin`，图标容器为 48–64 px 正圆。

## 7. 交互与动效

| 场景 | 参数 | 结果 |
| --- | --- | --- |
| 通用过渡 | 260 ms / `cubic-bezier(.2,.7,.2,1)` | 克制、稳定 |
| Hero 圆盘入场 | 720 ms / `cubic-bezier(.16,1,.3,1)` | 0.965 → 1，透明度渐入 |
| Hero 内容入场 | 560 ms | Y 16 px → 0，分段延迟 100–350 ms |
| 主按钮 Hover | 260 ms | Y -4 px，亮度 1.06 |
| 卡片 Hover | 260 ms | Y -2～-3 px，边框提亮 |
| 人格光晕 | 2600 ms / alternate | 0.96 → 1.035 循环呼吸 |
| 问卦 Loading | 720–960 ms | 六点轨道旋转后退出 |
| Toast | 2200 ms | 底部出现后自动消失 |

系统开启“减少动态效果”时：停止自动视频、移除循环呼吸、动画时长降至 0.01 ms、关闭平滑滚动和高成本模糊。

## 8. 响应式规范

- `> 900 px`：完整桌面导航，问卦双栏，工具四列。
- `761–900 px`：缩小容器内边距，工具两列，主结构保持桌面逻辑。
- `≤ 760 px`：全屏移动导航；问卦双栏改为上下堆叠；人格图谱单列；记录模块图文上下堆叠；卦象一次显示三张；工具保持两列。
- 所有横向滑轨启用 scroll snap，不显示滚动条，但必须保留触摸滑动能力。
- 不允许出现页面级横向滚动、标题遮挡、按钮文字折行或图像拉伸。

### 9.3 安全区标注

- 观象视频：人物、道路或建筑主体必须放在画面中央直径 68% 的安全圆内；顶部 12% 预留标题；底部 12% 预留播放按钮。
- 日卦山水：左侧 48% 为文字安全区，避免高对比物体；右侧 52% 可承载山峰、树木与卦象。
- 人格测试图：四周至少保留 32 px 安全边距，不将核心文字贴近圆角。
- 记录图：桌面端左侧约 24% 会叠加暗色渐变，核心视觉不要放在最左侧。
- 社交封面：所有关键信息保持在 1056 × 486 px 中央安全区。

### 9.4 命名与导出

- 文件名使用小写英文加连字符：`module-purpose-v2.webp`。
- 页面照片和纹理优先 WebP；需要透明通道时使用 PNG；图标使用 SVG；视频使用 MP4/H.264。
- 常规位图交付 1× 与 2×；超大背景按实际最大显示宽度导出，避免无意义 4×。
- 不在位图中烘焙按钮、描边、阴影、圆角和可编辑文字。
- 同一素材修改构图时升级版本号，不覆盖已上线文件。

## 10. Figma 标注与组件命名

建议页面：`00 Cover`、`01 Foundations`、`02 Components`、`03 Desktop`、`04 Mobile`、`05 Motion`、`06 Assets`。

组件命名示例：

- `Navigation/Header/Desktop`
- `Navigation/Header/Mobile`
- `Button/Primary/Default|Hover|Pressed|Loading`
- `Chip/Topic/Default|Selected`
- `Card/Question/Default|Hover`
- `Card/Tool/Default|Hover`
- `Media/OracleOrbit/Playing|Paused`
- `Feedback/Toast`
- `Feedback/QuestionLoader`

所有 Auto Layout 组件必须保留可变文字宽度；设计稿中的切图层统一以 `CUT-编号_用途` 命名，并在右侧属性说明中写入尺寸、倍率与裁切方式。

## 11. 无障碍与内容边界

- 正文与背景对比度目标不低于 WCAG AA；弱金色仅用于辅助信息。
- 可交互元素必须有可见焦点，触摸目标建议不小于 40 × 40 px。
- 图标按钮必须提供中文 `aria-label`；装饰图标对读屏隐藏。
- 视频必须有暂停入口，并响应减少动态效果设置。
- 产品文案避免“保证结果”“绝对吉凶”等表述；所有解读必须保留“仅供参考、重在自省、不替代现实决策”的边界。

## 12. 离线单文件交付

运行 `npm run build:single` 后生成 `release/东方观星-离线分享版.html`。该文件已内嵌 React 运行代码、CSS、页面图片和视频，可直接发送给他人并用现代浏览器双击打开。社交分享元信息不影响离线页面展示。
